# Исследование алгоритмов поиска подстрок

## Данное исследование выполнено на устройстве MacBook Air M1 2020. Код написан на языке Java 21. Для данного исследования я выбрал следующие тексты: 
1. Русский текст о профессии программиста (633 символа)
2. Английский текст книги "Властелин колец: Две крепости" (835 714 символов)
3. Английский текст об алгоритмах поиска (3435 символов)
4. Английский текст книги "Побег из Шоушенка" (174 288 символов)


### Основные алгоритмы:
1. **Наивный алгоритм** - базовый алгоритм полного перебора
2. **Рабин-Карп** с тремя хеш-функциями:
   - `SimpleHash` (полиномиальный хеш, base=256)
   - `HornerHash` (полиномиальный хеш, base=131) 
   - `XORHash` (побитовый XOR)
3. **Бойер-Мур** - алгоритм с эвристиками плохого символа
4. **Кнут-Моррис-Пратт (КМП)** - алгоритм с префикс-функцией



### Для всех тестов я выбрал единый паттерн "substring". В ходе работы я буду прогонять каждый тест по 10 раз, чтобы на основе полученных значений построить графики, на которых видно, как каждый из алгоритмов ведет себя с разными текстами. Также для алгоритма Рабина-Карпа будут фиксироваться коллизии хешей.
